# Arthas Command Guide

## 核心契约

- 默认输出 **1 条**最终命令，且必须单行紧凑。
- 默认优先观测命令；不主动输出改值/热更新。
- 默认按 CARD 模板生成；CARD 能满足时，不得改写为自定义 OGNL。
- 常规场景禁用复杂 OGNL（集合投影/多变量拼接/多段三元表达式）。
- 除非用户明确要求，否则不输出分段解释。

## 执行流程（最小上下文）

1. 识别模式：`问题驱动` / `命令驱动`。
2. 提取关键槽位：类名、方法名、过滤条件、目标字段、执行方式。
3. 先查 `references/arthas.md` 的 `6. 检索索引`，定位 `1~2` 个 CARD，再读取对应命令段。
4. 问题驱动按 `SCENE_ID` 路由；命令驱动遵循最小改动。
5. 通过质量/风险门禁后输出 1 条命令；仅在用户要求时附解释或收敛信息。

## 命令生成优先级（强制）

1. **CARD 原样优先**：直接使用 CARD 原始模板，仅替换槽位值。
2. **CARD 同家族最小改动**：仅调整 `类/方法/条件/-n/-x/事件点`，不切家族。
3. **OGNL 兜底**：仅在现有 CARD 无法表达目标时，才允许引入 OGNL。
4. **复杂 OGNL 需显式要求**：未被用户明确要求时，禁止输出复杂 OGNL。

## 决策规则

1. 风险分级：
   - 观测类：直接输出命令。
   - 高开销类（大范围 `trace/watch`、`heapdump`、`profiler/jfr`）：给 1 行轻提示，不二次确认。
   - 高风险类：先确认模板，再给命令（详单见 `references/arthas.md` 的 `5.1 风险触发映射`）。
2. 槽位缺失先澄清：缺类/方法/过滤目标时，不直接给命令。
3. 命令驱动最小改动：保留原家族与事件点，仅改用户指定项。
4. 家族切换需授权：原家族无法满足目标时，先说明并征得允许。
5. 表达式升级控制：CARD 满足时禁止升级为自定义 OGNL。

## 路由优先级（精简）

- `<ClassName> <methodName>` 优先方法级 CARD；问题驱动且方法级不可用时可回退 `WATCH_DISPATCHER_*`。
- 约束条件按 `header/url/query/body` 动态拼接，并统一空值保护。
- `watch` 问题驱动默认 `-f`；命令驱动默认保留原事件点。
- `Unknown Source/动态代理` -> `JAD_PROXY_METHOD`
- `运行时配置值/配置来源` -> `SPRING_CONFIG_VALUE` / `SPRING_CONFIG_SOURCE`
- `StackOverflowError 调用链不清` -> `STACK_DEPTH_GUARD`
- `静态字段/跨类加载器` -> `GETSTATIC_READ_FILTER`（`sc -d` 后 `ognl -c`）
- `集合过滤` -> 返回表达式与条件表达式分离，条件建议 `.size()>0` 收敛。

## 质量门禁

- 范围：优先类+方法，无法精准时必须有过滤条件。
- 成本：`watch/trace/tt` 必带 `-n`；`-x` 默认 `2`，按需提升到 `3+`。
- 条件：必须空值保护；不生效时优先 `WATCH_CONDITION_VERBOSE`（`-v`）。
- 表达式：默认使用 CARD 里的稳定表达式；禁止无必要的复杂 OGNL 拼接。
- 收敛：增强类操作需内部可回滚校验；默认不额外输出第二条命令。
- 风险：命中高风险触发项必须先确认。

## 输出模板

```text
<cmd>
```

## 可选扩展模板（仅用户明确要求解释时）

```text
命令：<cmd>
说明：<一句话>
收敛：<reset/stop/无>
```
