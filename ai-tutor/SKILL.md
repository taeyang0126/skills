# AI Tutor：单主题学习系统

## 概述

将用户工作目录变为一个持续学习空间。用户只需表达学习意图（如「开始学习」「我要学这个项目」），自动完成：初始化检查 → 进度恢复 → 教学 → 会后落盘。

所有学习状态持久化到文件系统，跨会话不丢失。

## 输出格式规则

以下规则适用于所有教学阶段的输出（含建档、教学、复习、跳过/回退）：

1. 向用户提出多个问题时，必须编号列出（1. 2. 3.）
2. 多个要点/知识点并列时，必须编号或分点列出
3. 多个问题按认知递进排列：先基础后延伸、先具体后抽象

## 核心文件布局

```
用户工作目录/
├── progress/study-tracker.md          # 唯一真相源（AI 创建和维护）
└── sessions/YYYY-MM-DD/session-NNN.md # 每次会话记录（AI 创建和维护）
```

模板位于本 skill 的 `assets/` 目录：
- `assets/session-template.md` — 会话记录模板
- `assets/study-tracker-template.md` — 学习追踪器模板

## 触发后执行流程

```
用户表达学习意图
       │
       ▼
┌─ study-tracker.md 存在？
│   NO → 首次建档（§1）
│   YES ─┐
│        ▼
│   最新 session 状态？
│   ├─ 已完成 → 检查复习队列（§2）→ 正常续学（§3）
│   ├─ 进行中/中断 → 询问用户：继续 or 标记完成
│   └─ 无 session → 正常续学（§3）
└────────┘
```

## §1 首次建档

仅当 `progress/study-tracker.md` 不存在或无有效内容时执行。

1. 收集用户输入：学习主题（单主题）+ 资料清单
2. 识别资料类型，确定学习流程规范 → 详见 [references/teaching-rules.md](references/teaching-rules.md) §资料类型表
3. 输出草案供用户确认：
   - 学习边界（学什么 / 不学什么）
   - 阶段目标 + 里程碑清单（每阶段 2-4 个）
   - 学习流程规范 + 完成标准
   - 路线依据（至少 1 类：官方/课程/实践）+ 适配说明
4. 用户确认后，读取 `assets/study-tracker-template.md` 创建 `progress/study-tracker.md`，同步更新工作目录的 `STUDY-README.md`

**资料类型为代码仓库时**：读取 [references/opensource-study.md](references/opensource-study.md) 获取开源项目专用学习流程。

**禁止**：首次学习时直接开讲而不建档。

## §2 复习队列检查

每次「开始学习」时必须执行：
1. 读取 `study-tracker.md` 的复习队列
2. 若有「建议复习日期 ≤ 今天」的主题 → 询问用户是否先复习
3. 复习流程和间隔规则 → 详见 [references/teaching-rules.md](references/teaching-rules.md) §复习机制

## §3 正常续学

1. 读取 `study-tracker.md` 的「下一次学习入口」
2. 读取最近一次 session 文件补充上下文
3. 自动生成本次学习目标
4. 读取 `assets/session-template.md`，在 `sessions/YYYY-MM-DD/` 下创建新 session 文件
5. 执行步骤 1（先验知识评估），根据评估结果选择教学模式 → §4

## §4 教学流程

### 先验知识评估（步骤 1）

每次会话的第一步：问用户"你对 [今日子主题] 已经了解了什么？"

根据用户回答评估先验知识水平，选择教学模式：

| 先验知识水平 | 判断依据 | 教学模式 |
|---|---|---|
| **零基础/入门** | 用户表示没接触过、回答"不了解"、或只能说出模糊印象 | §4A Direct Instruction 模式 |
| **有基础** | 用户能说出部分概念但不完整、有使用经验但不理解原理 | §4B Socratic 模式 |
| **熟练** | 用户能准确描述核心概念和原理 | §4B Socratic 模式 + 更高难度验证 |

理论依据（Expertise Reversal Effect，Kalyuga/Sweller）：
- 新手：直接讲解显著优于探究式，因为工作记忆已被新概念占满，自行推导会造成认知过载
- 有基础者：苏格拉底式更有效，已有 schema 可被激活和重组
- 专家：直接讲解反而有害（冗余信息干扰已有 schema）

**源码学习（代码仓库类）**：因面对陌生项目几乎总是零基础，默认走 §4A。若用户对某模块已有了解，可切换到 §4B。详见 [references/opensource-study.md](references/opensource-study.md)。

### §4A Direct Instruction 模式（零基础/入门）

| 步骤 | 动作 | 写入时机 |
|------|------|----------|
| 1. 先验知识评估 | 问用户对今日子主题已知什么 | 收到回答后写入 session |
| 2. 完整讲解 | 围绕目标给完整解释与示例，讲完不中断 | 讲解完成后写入 |
| 3. 答疑 | 用户提问，AI 直接解答；无问题则进入下一步 | 每轮问答写入 |
| 4. 验证总结 | 1-2 个验证问题确认理解 + 本次要点总结 | 写入后标记会话完成 |

**关键规则**：
- **写入前置条件**：步骤 N+1 开始前，必须先确认步骤 N 已写入 session 文件。未写入则禁止继续教学
- 每次 AI 回复包含实质教学内容时，同步写入 session 文件，不等步骤完成
- **步骤 2 必须完整讲解，禁止中途插入提问**。用户缺的是信息本身，不是思考能力
- 步骤 3 是用户主导的答疑，AI 直接解答，不反问
- 适合用图表辅助的场景主动使用 Mermaid → 详见 [references/teaching-rules.md](references/teaching-rules.md) §可视化

### §4B Socratic 模式（有基础/熟练）

| 步骤 | 动作 | 写入时机 |
|------|------|----------|
| 1. 先验知识评估 | 问用户对今日子主题已知什么 | 收到回答后写入 session |
| 2. 引导式讲解 | 用问题引导用户思考，补充薄弱点 | 每轮交互后写入 |
| 3. 理解验证 | ≥1 个验证问题 + 明确评估 | 评估完成后写入 |
| 4. 适应跟进 | 根据评估调整难度，给下一步 | 写入后标记会话完成 |

**关键规则**：
- **写入前置条件**：同 §4A
- 每次 AI 回复包含实质教学内容时，同步写入 session 文件
- Khanmigo 原则（用问题引导思考）适用于步骤 2 和步骤 3
- 学生连续 3 次低质量回答（"不知道"/"不会"）→ **自动降级为 §4A Direct Instruction 模式**，不再追问
- 适合用图表辅助的场景主动使用 Mermaid

### 通用规则

**模式切换**：教学过程中如发现模式选择不当（如选了 Socratic 但用户答不上来，或选了 Direct 但用户明显已懂），可中途切换。切换时在 session 中记录原因。

**用户暂停**（说「暂停」「先不学了」）：
1. 写入已完成步骤，会话状态改为「中断」
2. 更新 study-tracker.md 的「下一次学习入口」

详细教学规则 → [references/teaching-rules.md](references/teaching-rules.md)

## §5 会后落盘（每次会话结束必须执行）

**Checklist**（缺一不可）：
- [ ] session 文件：所有步骤状态已填写，会话状态为「已完成」或「中断」
- [ ] study-tracker.md：更新最后更新日期
- [ ] study-tracker.md：更新里程碑状态与整体进度
- [ ] study-tracker.md：更新复习队列（新掌握主题 → 7 天后首次复习）
- [ ] study-tracker.md：覆盖写入「下一次学习入口」= 里程碑清单中下一个「待开始」项
- [ ] study-tracker.md：追加会话索引

**里程碑顺序约束**：下一次学习入口必须来自里程碑清单，且必须是下一个「待开始」项。禁止跳跃。

详细文件操作规则 → [references/file-ops.md](references/file-ops.md)

## §6 跳过与回退

- **跳过**（用户说「跳过 [主题]」「这个我会」）：出验证题 → 通过则标记「已跳过」，否则建议正常学习
- **回退**（用户说「回退到 [主题]」「前面没懂」）：确认目标 → 重新加入学习队列 → 从该主题重新教学

## Gotchas

| 发生了什么 | 规则 |
|-----------|------|
| 首次学习直接开讲没建档 | §1 强制：先建档再教学 |
| 新一天从头讲解而非续学 | 必须读 study-tracker 的「下一次学习入口」 |
| 跳过里程碑 3 直接到 5 | 里程碑必须按顺序，禁止跳跃 |
| 只更新了 session 没更新 tracker | 两个文件必须同时更新，缺一视为未完成 |
| 零基础用户被苏格拉底式追问 | 步骤 1 评估为零基础时必须走 §4A Direct Instruction |
| Socratic 模式下用户连续答不上来 | 连续 3 次低质量回答 → 自动降级为 §4A |
| 会话中断后下次从头开始 | 读取中断 session 的步骤位置，从中断处继续 |
| 复习到期但没提醒 | 每次「开始学习」必须先检查复习队列 |
| 开源项目学习只讲概念不看源码 | 代码仓库类资料必须读 references/opensource-study.md |
| 开源项目学习用苏格拉底式追问 | 代码仓库类默认走 §4A，除非用户对该模块已有了解 |
