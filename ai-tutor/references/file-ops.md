## 目录结构约定

```
用户工作目录/
├── STUDY-README.md                    # 学习说明 + 资料基线（AI 维护，不覆盖项目原有 README.md）
├── progress/
│   └── study-tracker.md               # 学习追踪器（AI 创建和维护）
├── sessions/
│   └── YYYY-MM-DD/
│       ├── session-001.md
│       ├── session-002.md
│       └── ...
└── [用户学习资料]
```

禁止要求用户手动复制模板。所有文件由 AI 自动创建。

## 初始化检查流程

每次「开始学习」时按顺序执行：

### 1. 检查 study-tracker

- `progress/study-tracker.md` 不存在 → 按 `assets/study-tracker-template.md` 创建
- 存在但无有效内容 → 视为首次学习

### 2. 检查当天 session 目录

- `sessions/YYYY-MM-DD/` 不存在 → 创建目录 + `session-001.md`
- 目录存在但无 session → 创建 `session-001.md`
- 目录存在且有 session → 检查最新（序号最大）session 的状态

### 3. 会话状态判断

读取最新 session 的「会话状态」字段：

| 状态 | 行为 |
|------|------|
| 已完成 | 创建新 session（序号+1），正常续学 |
| 进行中 / 中断 | 询问用户：继续中断处 or 标记完成学新内容 |
| 不存在 | 首次学习，进入建档流程 |

## 会话文件管理

### 命名规则

- 路径：`sessions/YYYY-MM-DD/session-NNN.md`
- NNN 三位数补零：001, 002, 003...
- 同一天多次学习按序号递增

### 创建方式

读取 `assets/session-template.md` 模板内容，填入：
- 日期、学习主题
- 承接来源（来自 study-tracker 的「下一次学习入口」）
- 今日学习目标（基于里程碑自动生成）
- 会话状态初始为「进行中」

## 会后落盘规则

每次会话结束后，**必须同时**更新两个文件：

### session 文件更新

- 确认苏格拉底四步每步状态已填写
- 会话状态改为「已完成」或「中断」
- 填写本次产出（已掌握 / 新增缺口）

### study-tracker 更新

- 更新「最后更新」日期
- 更新里程碑状态与整体进度
- 更新知识缺口状态
- 更新复习队列（新掌握主题加入，设 7 天后复习）
- **覆盖写入**「下一次学习入口」= 里程碑清单中第一个「待开始」项
- 追加会话索引（链接到当前 session 文件）

**若只更新其一，视为流程未完成。**

### 会后更新检查清单

写入 study-tracker 末尾：
- [ ] 更新最后更新日期
- [ ] 更新里程碑状态与整体进度
- [ ] 更新缺口状态
- [ ] 更新复习队列
- [ ] 覆盖写入「下一次学习入口」（必须来自里程碑清单且为下一个待开始项）
- [ ] 追加本次会话索引

## 里程碑顺序约束

**核心原则**：「下一次学习入口」必须来自里程碑清单，且必须按顺序推进。

### 规则

1. 下一步主题 = 里程碑清单中第一个「待开始」的里程碑
2. 禁止跳过中间里程碑
3. 只有用户触发「跳过」规则并验证通过时，才能标记「已跳过」并继续下一个

### 校验流程（每次会话结束时执行）

1. 读取里程碑清单
2. 找到第一个「待开始」的里程碑
3. 将其作为「下一次学习入口」写入 study-tracker
4. 若找不到「待开始」项 → 本阶段已完成，与用户确认是否进入下一阶段或结束

---

## 进度计算规则

```
整体进度 = 已完成里程碑数 / 总里程碑数 × 100%
```

特殊规则：
- 进行中的里程碑按 50% 计入
- 已跳过的里程碑不计入已完成，但计入总数

## STUDY-README 同步规则

`STUDY-README.md` 维护主题与资料基线（使用独立文件名，避免覆盖项目原有 README.md）。触发同步的场景：
1. 首次建档完成后
2. 用户新增/删除学习资料后
3. 用户调整学习边界或阶段目标后

每次同步至少更新：
- 学习主题
- 资料清单
- 学习边界
- 阶段目标
- 路线依据与适配说明
- 最近更新时间

README 中的「学习主题与资料基线（AI 维护）」区块结构：

```markdown
## 学习主题与资料基线（AI 维护）

- 学习主题：[主题]
- 学习边界（学什么）：[边界]
- 非学习边界（暂不学什么）：[排除项]
- 阶段目标：
  1. [目标 1]
  2. [目标 2]
- 路线依据：[类型 + 说明]
- 适配说明：[为什么适合]
- 资料清单：
  1. [资料 1]
  2. [资料 2]
- 最近更新时间：YYYY-MM-DD
```

## 跨天续学规则

新的一天开始学习时，必须延续前一天进度：

1. **优先**使用 study-tracker 的「下一次学习入口」
2. 其次参考"进行中里程碑"和"未解决缺口"
3. **禁止**默认从头讲解

只有用户明确说「重置学习」「从头开始」时，才允许重排路线。

---

## 最小完成定义（DoD）

一次学习会话完成，需同时满足：

1. 用户完成本次理解验证问题
2. 苏格拉底四步每步都已写入 session
3. 会话状态已更新（已完成 / 中断）
4. tracker 已更新：里程碑状态、整体进度、复习队列、下一次学习入口
