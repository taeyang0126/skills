## 资料类型表

首次建档时，根据资料类型确定学习路线拆分方式和流程规范：

| 资料类型 | 识别特征 | 拆分方式 | 学习流程 |
|----------|----------|----------|----------|
| 官方文档 | 官方域名、docs/ | 按文档目录结构 | 概念讲解 + 实践示例 |
| 论文/学术文章 | PDF、arXiv、DOI | Abstract → 背景 → 核心理论 → 实验 → 结论 | 理论讲解 + 案例分析 |
| 书籍/教材 | 章节结构、ISBN | 按章节，每章 1-2 个会话 | 概念讲解 + 习题练习 |
| 视频课程 | YouTube、Coursera、B站 | 按课时/章节 | 观看学习 + 笔记整理 |
| 代码仓库 | GitHub/GitLab | 先 README/架构，再按模块 | 理论讲解 + 源码解读 |
| 研报/行业报告 | 证券研报、白皮书 | 核心结论 → 论证 → 数据 | 结论解读 + 论证分析 |
| 法规/政策文件 | 法律条文、标准规范 | 核心条款 → 适用范围 → 实务 | 条款解读 + 应用场景 |
| 财报/数据报告 | 年报、季报 | 核心指标 → 趋势 → 结论 | 指标解读 + 趋势分析 |
| 博客/文章 | 博客平台、单篇 | 作为补充材料，不作主线 | 快速阅读 + 要点提取 |

## 教学步骤详细规则

### 步骤 1：先验知识评估（两种模式共用）

提问："你对 [今日子主题] 已经了解了什么？"

目的：评估先验知识水平，决定使用 Direct Instruction（§4A）还是 Socratic（§4B）模式。记录用户原话。

判断标准见 SKILL.md §4。

### §4A Direct Instruction 模式详细规则

适用于：零基础/入门学习者（步骤 1 评估结果）。

**步骤 2：完整讲解**

围绕今日目标给出完整解释与示例。规则：
- 根据 study-tracker 中的「学习流程规范」组织内容
- **必须完整讲解，禁止中途插入提问**
- 适合用图表的场景主动使用 Mermaid（见下方§可视化）
- 不做与主题无关的扩展

**步骤 3：答疑**

- 讲解完成后询问用户是否有疑问
- 用户提问时**直接解答**，不反问
- 用户无问题则直接进入步骤 4

**步骤 4：验证总结**

- 1-2 个验证问题确认理解
- 评估结果必须明确：已理解 / 部分理解 / 未理解
- 输出本次学习要点总结

### §4B Socratic 模式详细规则

适用于：有基础/熟练学习者（步骤 1 评估结果）。

**步骤 2：引导式讲解**

用问题引导用户思考，补充薄弱点。规则：
- 根据 study-tracker 中的「学习流程规范」组织内容
- Khanmigo 原则：用问题引导思考，不直接给答案
- 适合用图表的场景主动使用 Mermaid
- 不做与主题无关的扩展

**步骤 3：理解验证**

- 至少 1 个验证问题
- 评估结果必须明确：已理解 / 部分理解 / 未理解
- 评估依据：用户能否用自己的话解释核心概念，而非复述原文

**步骤 4：适应跟进**

根据评估结果：
- **已理解**：确认掌握，给出下一步方向
- **部分理解**：补充薄弱点，可追加 1 个针对性问题
- **未理解**：换个角度重新解释，降低难度

### 实时写入规则

**核心原则**：每次 AI 回复包含实质教学内容时，同步写入 session 文件。步骤 N+1 开始前，必须先确认步骤 N 已写入，未写入则禁止继续教学。

每完成一步，**立即**写入 session 文件对应区块：
- 步骤 1 完成 → 写入 AI 提问 + 学生回答 + 评估的先验知识水平 + 选择的教学模式
- 步骤 2 完成 → 写入核心知识点 + 状态改为「已完成」
- 步骤 3 完成 → 写入问题 + 回答 + 评估结果 + 状态改为「已完成」
- 步骤 4 完成 → 写入跟进动作/要点总结 + 状态改为「已完成」+ 会话状态改为「已完成」

## 防帮助滥用机制（仅适用于 §4B Socratic 模式）

借鉴 Khan Academy Khanmigo 的设计。

**低质量回答检测**：用户连续回答"不知道""不会""IDK"或其他无实质内容的回答。

**处理流程**：
1. 第 1 次：正常引导，拆解问题为更小的部分
2. 第 2 次：提供选项列表让用户选择，降低回答门槛
3. 第 3 次：**自动降级为 §4A Direct Instruction 模式**，在 session 中记录降级原因，从完整讲解开始

**例外**：对于纯事实性知识（定义、日期、公式），如果用户确实不知道，可以提供选项列表供选择，而非强制追问。

## 跳过与回退规则

### 跳过（用户已掌握）

触发词：「跳过 [主题]」「这个我会」「这个不用学了」

流程：
1. 出 1 个验证问题确认掌握程度
2. 评估：
   - **已理解** → 标记「已跳过（验证通过）」，更新里程碑
   - **部分理解** → 建议快速过核心点，用户决定
   - **未理解** → 建议正常学习，用户决定
3. 更新 session 和 tracker

### 回退（前面没学懂）

触发词：「回退到 [主题]」「前面 [主题] 没懂」「重新学 [主题]」

流程：
1. 确认回退目标（具体到哪个里程碑）
2. 将该主题重新加入学习队列，状态改为「进行中」或「待开始」
3. 从该主题重新开始教学
4. 后续已完成的里程碑标记为「需巩固」

## 复习机制

### 间隔规则（简化 SM-2）

| 复习次数 | 间隔 |
|----------|------|
| 首次 | 7 天后 |
| 第 2 次 | 14 天后 |
| 第 3 次 | 30 天后 |
| 第 4 次+ | 90 天后 |

复习未通过 → 间隔重置为 7 天，重新学习。

### 复习流程

1. **快速回顾**：简要总结该主题核心点
2. **验证问题**：1-2 个问题检验记忆
3. **评估**：通过 → 延长间隔；未通过 → 重置间隔

### 新增复习项

每次会话中「已掌握」的主题：
1. 添加到复习队列
2. 设置首次复习日期 = 当天 + 7 天

## 可视化辅助（Mermaid）

### 适用场景

遇到以下内容时**主动使用** Mermaid 图表：
- 流程/步骤：多步操作、决策流程、数据流转
- 架构/结构：系统架构、模块关系、层次结构
- 时序/交互：时序图、消息传递
- 状态转换：状态机、生命周期
- 关系/对比：概念关系图、思维导图

用户也可主动触发：「画个图」「用图说明」。

### 图表类型选择

不限制类型，根据内容选最合适的：flowchart、sequenceDiagram、stateDiagram-v2、classDiagram、erDiagram、gantt、pie、mindmap。

### 输出位置

- **嵌入式**：在步骤 2 的知识点中直接嵌入
- **独立章节**：在 session 文件的「可视化辅助」章节集中存放
- 图表与单个知识点紧密相关 → 嵌入；概括整个会话 → 独立章节

### 写入规范

每次使用 Mermaid：
1. 图表前有简要说明（展示什么）
2. 使用正确的 mermaid 代码块语法
3. 图表后可选文字解读
